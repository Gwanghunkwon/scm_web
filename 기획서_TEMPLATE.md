# 경량 SCM 시스템 기획서 (MVP 최신)

## 1. 프로젝트 개요
- 목적: 제품/BOM/재고/생산계획 입력을 기반으로 부족 원재료와 발주량을 자동 계산
- 사용자 가치: "지금 재고로 무엇이 얼마나 부족한지"를 즉시 확인
- 배포 채널: Vercel 프론트 + 공개 FastAPI 백엔드

## 2. 핵심 기능 범위
- 품목 관리: 제품(PRODUCT) / 원재료(RAW) 등록
- BOM 관리: 수동 입력 + 엑셀(.xlsx/.xls) 업로드
- 재고 관리: 재고 스냅샷 등록(창고는 MVP에서 MAIN 자동 처리)
- 생산계획: 기간별 계획 등록
- 계산: 대시보드(`GET /api/dashboard`) + 다제품 계산(`POST /api/calculate`)

## 3. 사용자 시나리오
1) 품목 등록(제품/원재료)
2) BOM 입력 또는 엑셀 업로드
3) 재고 입력(품목/수량/기준일)
4) 생산계획 입력
5) 대시보드에서 기간 선택 후 자동 계산 확인

## 4. UI/UX 원칙
- 상단 메뉴: 대시보드 / 품목 / 배합비(BOM) / 재고 / 생산계획
- 입력 최소화: 창고 선택 제거, 재고는 기본 창고 자동 처리
- 장애 안내: API 미연결 시 원인과 조치(NEXT_PUBLIC_API_URL) 즉시 표시

## 5. 데이터/계산 원칙
- 필요량 = Σ(생산량 × BOM)
- 부족량 = 필요량 - 현재재고
- 발주량 = max(부족량, 0)
- 재고 없음은 0으로 간주
- BOM 없는 제품은 계산 제외

## 6. 비기능 요구
- 배포 페이지 첫 화면 로딩 시 핵심 경고/안내 3초 이내 표시
- 계산 응답 실패 시 사용자 친화 에러 메시지
- Vercel 배포 시 env 누락을 문구로 즉시 가이드

## 7. 수용 기준

- [ ] 재고 등록이 창고 선택 없이 완료
- [ ] API 미연결 시 안내문으로 원인 파악 가능
