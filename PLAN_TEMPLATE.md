# PLAN

## 프로젝트 개요
- 프로젝트명: SCM MVP (BOM + 재고 기반 자동 발주)
- 한 줄 목표: 부족 원재료를 자동 계산하고 발주량을 대시보드로 즉시 확인
- 현재 단계: MVP 기능 구현 + Vercel 연동 안정화
- 최종 마감: 1차 운영 전환 준비

## 요구사항 재정의 (확정)
- 사용자는 웹에서 **제품/원재료, BOM, 재고, 생산계획**을 직접 등록한다.
- 대시보드는 `GET /api/dashboard`로 DB 데이터를 읽어 KPI/차트/부족리스트를 계산한다.
- 계산 API `POST /api/calculate`는 다제품 생산계획 배열을 지원한다.
- 창고는 사용자 입력을 단순화하기 위해 MVP에서 **기본 MAIN 창고 자동 사용**을 기본값으로 한다.
- Vercel 프론트는 반드시 `NEXT_PUBLIC_API_URL`(공개 백엔드)로 연결한다.

## 주요 리스크
- Vercel 환경변수 누락 시 `Failed to fetch` 발생
- 배포 브랜치/저장소 불일치 시 구버전 화면 노출
- 초기 데이터(제품/BOM/재고) 누락 시 대시보드 공백

## 단계별 실행 계획
### 1단계: 연결 안정화
- Vercel env 설정, CORS 확인, 최신 배포 검증
- 완료 기준: 품목 목록 조회/등록이 배포 주소에서 정상 동작

### 2단계: 입력 플로우 단순화
- 창고 메뉴 의존성 제거(재고 입력은 MAIN 자동 처리)
- 에러 메시지 사용자 친화화
- 완료 기준: 재고 화면에서 품목/수량/기준일만으로 저장 가능

### 3단계: 운영 준비
- 스펙/기획/계획/체크리스트 최신화
- 핵심 시나리오(품목→BOM→재고→대시보드) E2E 점검
- 완료 기준: 실사용자가 브라우저에서 전체 흐름 수행 가능

## 완료 기준 (DoD)
- [ ] 배포 사이트에서 품목 등록/조회가 정상 동작
- [ ] BOM 업로드/재고 입력 후 대시보드 계산 결과가 갱신
- [ ] 생산계획 자동 반영(ON/OFF) 동작 확인
- [ ] 운영 문서(환경변수, 배포, 장애대응) 최신화 완료
