# 프로젝트 템플릿 가이드

이 디렉토리에는 새 프로젝트를 시작할 때 사용할 템플릿 파일들이 있습니다.

## 📁 템플릿 파일 목록

### 1. `PLAN_TEMPLATE.md`
프로젝트 계획서 템플릿
- 프로젝트 개요
- 요구사항 재정의
- 주요 리스크
- 단계별 실행 계획
- 완료 기준

**사용 시점**: 프로젝트 시작 시, 또는 주요 단계 전환 시

### 2. `CONTEXT_TEMPLATE.md`
기술적 의사결정 기록 템플릿
- 시스템 구성
- 핵심 아키텍처 흐름
- 핵심 의사결정 기록 (ADR)
- 코딩/아키텍처 원칙
- 리스크 및 제약

**사용 시점**: 프로젝트 시작 시, 또는 중요한 기술 결정 시

### 3. `CHECKLIST_TEMPLATE.md`
진행 체크리스트 템플릿
- 현재 상태
- 진행 체크리스트 (기획/설계, 개발 환경, 프론트엔드, 백엔드, 테스트, 문서화, 배포)
- 이슈 트래킹
- 다음 액션 아이템

**사용 시점**: 프로젝트 시작 시, 작업 추적이 필요할 때

### 4. `기획서_TEMPLATE.md`
기능 명세서 템플릿
- 프로젝트 개요
- 기능 요구사항
- 사용자 시나리오
- UI/UX 설계
- 데이터 모델
- 기술 스택
- 수용 기준

**사용 시점**: 기능 개발 전, 명세가 필요할 때

### 5. `계획서_TEMPLATE.md`
구현 계획서 템플릿
- 프로젝트 범위
- 아키텍처 설계
- 데이터베이스 설계
- API 설계
- 데이터 흐름
- 단계별 구현 계획
- 필요한 리소스
- 리스크 및 대응 방안
- 테스트 계획
- 배포 계획

**사용 시점**: 구현 시작 전, 상세 계획이 필요할 때

## 🚀 사용 방법

### 방법 1: AI에게 직접 요청
```
"@PROJECT_START_TEMPLATE.md 기준으로 새 프로젝트 시작해줘. 
프로젝트명: [프로젝트명], 목표: [한 줄 목표]"
```

### 방법 2: 템플릿 파일 복사 후 수정
1. 필요한 템플릿 파일을 프로젝트 루트로 복사
2. 파일명에서 `_TEMPLATE` 제거 (예: `PLAN_TEMPLATE.md` → `PLAN.md`)
3. 내용을 프로젝트에 맞게 수정
4. AI에게 "@PLAN.md 업데이트해줘" 요청

### 방법 3: 기존 프로젝트 참고
- `IAP_기획서.md`, `IAP_계획서.md`, `IAP_체크리스트.md` 참고
- 실제 프로젝트에서 사용된 예시를 확인 가능

## 📋 템플릿 선택 가이드

### 작은 기능 추가/수정
- `CHECKLIST_TEMPLATE.md`만 사용

### 새로운 기능 개발
- `기획서_TEMPLATE.md` → `계획서_TEMPLATE.md` → `CHECKLIST_TEMPLATE.md` 순서

### 대규모 프로젝트
- 모든 템플릿 사용 권장
- `PLAN_TEMPLATE.md` → `CONTEXT_TEMPLATE.md` → `기획서_TEMPLATE.md` → `계획서_TEMPLATE.md` → `CHECKLIST_TEMPLATE.md`

## 💡 팁

1. **템플릿은 가이드라인**: 모든 항목을 채울 필요는 없습니다. 프로젝트에 필요한 부분만 사용하세요.

2. **지속적 업데이트**: 프로젝트 진행 중 `PLAN.md`, `CONTEXT.md`, `CHECKLIST.md`를 지속적으로 업데이트하세요.

3. **AI 활용**: 템플릿을 채울 때 AI에게 "이 항목을 채워줘"라고 요청하면 도움이 됩니다.

4. **버전 관리**: 중요한 변경사항은 `CONTEXT.md`의 ADR 섹션에 기록하세요.
